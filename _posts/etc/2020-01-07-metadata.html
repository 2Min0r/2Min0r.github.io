jekyll github metadata 경고 해결

Windows10 사용자입니다. 만약 Start Command Prompt with Ruby를 통해 bundle exec jekyll serve시 아래와 같은 경고메시지가 보일 때 해결하는 방법입니다. 참고사이트를 보고 windows의 경우 해결방법을 기록합니다.

GitHub Metadata: No GitHub API authentication could be found. Some fields may be missing or have incorrect data.

 

아래의 방법은 github-metadata을 통해 gem과 plugin을 설치하신 후 진행하시기 바랍니다.

 

1. Github personal access tokens

  1. 본인의 깃헙페이지의 오른쪽 위 프로필사진을 클릭합니다.

  2. Settings를 클릭합니다.

  3. 왼쪽 메뉴바 하단의 Develope settings를 클릭합니다.

  4. Personal access tokens를 클릭합니다.

  5. Generate new token을 클릭합니다.

  6. Note에 해당 토큰의 이름을 입력합니다. ex) token-for-terminal-app

  7. repo의 public_repo에 체크합니다.

  8. Generate token을 클릭합니다.

    ⭐️토큰이 생성되었으면, 해당 토큰을 복사합니다. 이 토큰은 페이지를 나가게 되면 더이상 볼 수 없습니다. 안전하게 토큰을 보관하며, 다른 사람에게 공유되지 않도록 해야 합니다.⭐️

 

2. 환경변수 생성

  1. 시작 > 검색 > cmd입력을 통해 명령 프롬프트 창을 켭니다.
  2. setx JEKYLL_GITHUB_TOKEN "[복사한토큰]" -m을 입력합니다. 이 방법은 참고사이트와 같은 방법으로도 똑같이 할 수 있습니다.
  3. 명령 프롬프트 창을 닫은 뒤 다시 켜서 echo %JEKYLL_GITHUB_TOKEN%을 입력하여 확인합니다.
  4. Start Command Prompt with Ruby 창을 다시 켠 후 위와 같은 방법으로 확인할 수 있으며, bundle exec jekyll serve를 입력하면 해당 경고가 사라집니다. (추가참고사이트)

 

혹시 잘못된 부분이나 궁금한 부분이 있으시면 아직 댓글창을 만들지 못해 블로그 하단의 SNS나 이메일로 보내주시기 바랍니다ㅎㅎ..😓
