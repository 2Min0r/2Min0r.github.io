  2. 프로그래머 첫걸음 11

    • 프로그래밍 패러다임

      • 객체지향 프로그래밍과 함수형 프로그래밍이 있다.

    • 함수형 프로그래밍

      • 전역상태를 사용하거나 변경하지 않음. 오직 자신에게 전달된 매개변수만 변경함

      • 부수효과(side effect)가 존재하지 않음

        • 현재 함수의 외부에 있는 데이터에 의존하지 않으며, 현재 함수의 외부에 있는 데이터를 변경하지 않음

      • 단점: 전역상태를 인정해야 쉽게 풀리는 문제도 있음

    • 객체지향 프로그래밍

      • 객체에 상태를 저장하는 방식으로 문제를 해결함

      • 클래스(비슷한 객체를 구분하거나 그룹으로 묶는 매커니즘)로 객체 만듦

      • 모든 객체는 클래스의 인스턴스

        • 객체와 인스턴스는 같은 용어

      • 클래스의 이름은 대문자로 시작하며, 여러 단어일 경우에는 대문자로 구분한다. ex) LikeThis (o), Like_this(x)

      • 매직 메서드(magic method): __init__처럼 밑줄 두 개를 쓴 메서드로 특별한 경우에 사용하는 메서드

      • 새 객체를 만드는 것 = 클래스를 인스턴스화한다.

        class Orange():
    def __init__(self, w, c):
        self.weight = w
        self.color = c
        print("created!")
