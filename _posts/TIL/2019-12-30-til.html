  2. 프로그래머 첫걸음 11, 12, 13

    • 객체지향프로그래밍

      • 장점: 코드 재사용이 용이하여 개발 및 관리 시간 효율적, 어떤 문제를 여러 조각으로 나누어 관리하기 쉬운 코드를 개발할 수 있음
      • 단점: 설계할 때 계획을 세우는 시간이 오래 걸림

    • 캡슐화

      • 변수와 메서드를 하나의 객체에 묶어야 한다.

      • 클라이언트가 클래스 내부 데이터에 직접 접근하지 못하도록 감춘다.

        • 클라이언트: 클래스 외부에서 객체를 사용하는 코드

      • 파이썬에는 프라이빗 변수가 없다.

        • 이름 규칙을 사용하여 프라이빗 사용함
        • 호출자가 접근해서는 안 되는 변수나 메서드 앞에 밑줄(_) 붙임
        • ex) self._weight, def _area(self): pass

    • 추상화

      • 무언가로부터 어떤 특징을 떼어 내거나 제거해서 핵심적인 특징만 남기는 작업

    • 다형성

      • 기반이 되는 형태(type)이 달라도 같은 인터페이스를 부여할 수 있음

    • pass keyword

      • 아무일도 하지 않는다.

      • class Pass(self):
        pass

    • 메서드 오버라이딩(method overriding)

      • 자식 클래스에서 부모 클래스의 메서드를 그대로 사용하지 않고 같은 이름으로 다른 메서드를 만드는 것

    • 합성(Composition)

      • 어떤 객체를 다른 객체의 변수로 저장함으로써 'has-a' 관계, 소유관계를 나타냄

    • 클래스 변수와 인스턴스 변수

      • 클래스 변수: 파이썬이 클래스 정의에 따라 만드는 객체, 그리고 그 클래스를 통해 만드는 객체

        • 전역변수에 의존하지 않고 클래스의 인스턴스 전체에서 데이터를 공유할 수 있음

      • 인스턴스 변수: self.[변수이름] = [변수값] 으로 정의하는 것으로 객체에 속함

    • 매직 메서드

      • __repr__: 파이썬은 객체를 출력할 때 Object에서 상속한 __repr__이라는 매직 메서드를 호출하고 이 메서드가 반환하는 것을 그대로 출력한다. 👉 __repr__메서드를 오버라이드하면 출력내용 변경가능

      • __add__: 덧셈연산자(+)의 피연산자로서 객체를 사용할 수 있게 정의할 수 있음.

        • ex) [객체] + [객체] = [정의한 결과]의 방식으로!

      • is keyword: 같은 객체를 가리킬 경우 True, 다른 객체를 가리킬 경우 False. 변수가 None인지 확인할 경우에도 사용 가능

        • print(bob is same_bob)
    print(bob is another_bob)

      • __lt__: lesser than 으로 a < b일 경우

      • __gt__: greater than으로 a > b일 경우

  1. Git commit message(참고사이트)

    • 오타수정: Fix typo

    • Update는 잘못된 것을 바로잡는 것이 아니라 정상적으로 동작하지만 수정, 추가, 보완한다는 개념. 코드보다는 주로 문서나 리소스, 라이브러리 등에 사용함

    • ADD는 코드나 테스트, 예제, 문서 등의 추가가 있을 때 사용함

      • 그럼 TIL을 Update 2019-12-30 TIL 형식으로 커밋메시지를 작성하고 있었는데, Add 2019-12-30 TIL으로 작성하는게 맞는 것 같다.

    • Commit message는 항상 제3자가 알아보기 쉽도록! 이해할 수 있도록!
